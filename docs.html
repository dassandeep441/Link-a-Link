Docs/Confluence Mapping/Overview

Confluence Mapping

Map old Data Center links to Cloud equivalents after migration. Find missing pages, search mappings, and export results.

What is Confluence Mapping?

When you migrate from Confluence DC to Cloud, page IDs change. Links referencing old DC page IDs break. Confluence Mapping lets you upload a mapping file that cross-references old IDs with Cloud equivalents.

Prerequisites

You need: (1) old DC page IDs from your database, and (2) the Cloud mapping file from Atlassian.

Key Capabilities

  • Bulk Upload — Import DC-to-Cloud CSV
  • Search — Find any old DC link instantly
  • Missing Detection — Identify failed migrations
  • CSV Export — Export for auditing
  • Dashboard — Stats overview
  • Admin Controls — Filter, manage, delete

Workflow

StepWhoAction
1AdminExtract old DC links via SQL
2AdminRequest Cloud mapping from Atlassian
3AdminUpload combined CSV
4UsersSearch old DC links
5UsersExport results
Docs/Confluence Mapping/SQL Queries

SQL Queries for DC Links

Extract old Data Center page IDs from your database.

PostgreSQL

SELECT c.CONTENTID AS dc_page_id, s.SPACEKEY AS space_key,
  c.TITLE AS page_title,
  CONCAT('/wiki/spaces/', s.SPACEKEY, '/pages/', c.CONTENTID) AS dc_link
FROM CONTENT c JOIN SPACES s ON c.SPACEID = s.SPACEID
WHERE c.CONTENTTYPE = 'PAGE' AND c.CONTENT_STATUS = 'current'
ORDER BY s.SPACEKEY, c.TITLE;
Important

Run on a read replica or during off-peak hours.

Docs/Confluence Mapping/Getting Cloud Links

Getting Cloud Links from Atlassian

Request the DC-to-Cloud mapping file after migration.

How to Request

  1. Go to support.atlassian.com
  2. Create request under Cloud Migration
  3. Request the DC-to-Cloud page ID mapping file
  4. Specify migration batch date
  5. Atlassian provides a CSV mapping old IDs to new IDs
Tip

Request immediately after migration. Atlassian may take 2-5 business days.

Uploading Mapping Data

Admin guide for uploading DC-to-Cloud mapping data.

Steps

  1. Open Link-a-Link Admin Panel
  2. Select Confluence Mapping tab

  5. Review file preview
  6. Click Confirm Upload

Docs/Smart URLs/Overview

Smart URLs (Interlink)

Centralized, dynamic URLs that auto-update across all pages.

How It Works

Admins define system name + base URL. Authors use a macro with prefix + document ID. URL is constructed at render time. Base URL changes propagate instantly.

Benefits

  • Single source of truth for external URLs
  • Zero manual updates on URL change
  • Consistent patterns across content
  • Organized in folders by system
  • Future-proof against migrations
Docs/Smart URLs/Adding Systems

Adding Systems & URLs

Configure systems and base URLs.

  1. Open Smart URLs in admin
  2. Click + Add System
  3. Enter system name
  4. Enter base URL
  5. Click Save
Auto-grouping

URLs for an existing system auto-group in that system folder.

Docs/Smart URLs/Changing URLs

Changing Base URLs

Update a base URL and propagate to all pages.

  1. Open Smart URLs admin
  2. Find the system folder
  3. Click edit on the base URL
  4. Enter new URL
  5. Click Save

Every page using that prefix immediately resolves to the new URL on next view. No delay, no background job.

Verify First

This affects all pages using this prefix. Ensure the new URL is correct.

Docs/Smart URLs/Macro

Using the Macro

Insert Smart URLs into Confluence pages.

  1. Edit a page, type /link-a-link
  2. Select Smart URL macro
  3. Choose system from dropdown
  4. Enter document ID
  5. Click Insert
No Re-edit Needed

Base URL changes don't require page edits. The macro resolves at view time.

Docs/Privacy/Data Handling

Data Handling Overview

What we store and what we don't.

Stored

  • Mapping records (DC + Cloud IDs)
  • Smart URL configs (system names + base URLs)

NOT Accessed

  • Confluence page content
  • User personal data
  • Analytics or tracking data
Docs/Privacy/Deleting Data

Deleting Your Data

Permanently delete all Link-a-Link data.

  1. Open Admin Panel
  2. Go to Settings → Data Management
  3. Click Delete All Data
  4. Confirm deletion
Permanent

All records are permanently removed. Cannot be reversed.
