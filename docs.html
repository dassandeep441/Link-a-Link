Confluence Mapping

Map old Data Center links to their Cloud equivalents after migration. Find missing pages, search mappings, and export results.

📸 Confluence Mapping — Admin panel for uploading and managing migration CSV data

What is Confluence Mapping?

When you migrate from Confluence Data Center to Cloud, page IDs change. Links that referenced old DC page IDs will break in Cloud. Confluence Mapping lets you upload a mapping file that cross-references old DC page IDs with their new Cloud equivalents.

Once the mapping data is loaded, any user with access to the plugin can search for old DC links and instantly find the corresponding Cloud page — or identify links that didn't migrate successfully.

ℹ Before You Start

You'll need two things: (1) the old DC page IDs from your database, and (2) the Cloud mapping file from Atlassian. See the Pre-requisites sections in the sidebar for step-by-step instructions.

Key Capabilities

  • Bulk Upload — Import your DC-to-Cloud mapping CSV through the admin panel
  • Search — Find any old DC link and get the corresponding Cloud page instantly
  • Missing Link Detection — Identify pages that failed to migrate
  • CSV Export — Export search results for auditing and reporting
  • Dashboard — View statistics on total mapped links, found vs. missing
  • Admin Controls — Filter, manage, and delete mapping data at any time

Workflow Overview

1AdminExtract old DC links via SQL queries
2AdminRequest Cloud mapping file from Atlassian
3AdminUpload combined mapping CSV to Link-a-Link
4UsersSearch for old DC links → get Cloud equivalents
5UsersExport results as CSV for reporting
Docs›Confluence Mapping›SQL Queries

SQL Queries for DC Links

Extract old Data Center page IDs and links from your Confluence DC database before migration.

PostgreSQL

Run the following query against your Confluence Data Center PostgreSQL database to extract page IDs with their space keys and titles:

SELECT
  c.CONTENTID     AS dc_page_id,
  s.SPACEKEY      AS space_key,
  c.TITLE         AS page_title,
  CONCAT('/wiki/spaces/', s.SPACEKEY,
         '/pages/', c.CONTENTID) AS dc_link
FROM CONTENT c
JOIN SPACES s ON c.SPACEID = s.SPACEID
WHERE c.CONTENTTYPE   = 'PAGE'
  AND c.CONTENT_STATUS = 'current'
ORDER BY s.SPACEKEY, c.TITLE;
⚠ Important

Run this query on a read replica or during off-peak hours to avoid impacting production performance.

MySQL / MariaDB

SELECT
  c.CONTENTID     AS dc_page_id,
  s.SPACEKEY      AS space_key,
  c.TITLE         AS page_title,
  CONCAT('/wiki/spaces/', s.SPACEKEY,
         '/pages/', c.CONTENTID) AS dc_link
FROM CONTENT c
INNER JOIN SPACES s ON c.SPACEID = s.SPACEID
WHERE c.CONTENTTYPE   = 'PAGE'
  AND c.CONTENT_STATUS = 'current'
ORDER BY s.SPACEKEY, c.TITLE;

dc_page_idspace_keypage_titledc_link
1024ENGAPI Reference/wiki/spaces/ENG/pages/1024
2048HROnboarding Guide/wiki/spaces/HR/pages/2048

Export this output as a CSV file. You'll combine it with the Cloud mapping file in the upload step.

Getting Cloud Links from Atlassian

How to request the DC-to-Cloud page ID mapping file from Atlassian Support after your migration.

Why You Need This

During migration, Atlassian assigns new page IDs in Cloud. The mapping file connects old DC page IDs to their new Cloud page IDs. Without it, there's no programmatic way to resolve old links.

How to Request

  1. Go to Atlassian Support (support.atlassian.com)
  2. Create a new support request under Cloud Migration
  3. Mention you need the DC-to-Cloud page ID mapping file
  4. Specify the migration batch date if applicable
  5. Atlassian will provide a CSV mapping old to new IDs
💡 Tip

Request this file immediately after migration. Atlassian may take 2–5 business days to provide it.

Expected File Format

dc_content_idcloud_content_idcloud_url
10245012/wiki/spaces/ENG/page/5012
20487834/wiki/spaces/HR/page/7834

Uploading Mapping Data

Step-by-step guide for admins to upload DC-to-Cloud mapping data into Link-a-Link.

📸 The CSV upload interface in the Confluence Mapping admin panel

Prerequisites

  • Confluence Admin permissions required
  • Mapping CSV file prepared (see previous sections)
  • Link-a-Link installed on your instance

Upload Steps

  1. Open the Link-a-Link Admin Panel in Confluence
  2. Click Confluence Mapping in the left sidebar
  3. Click "Choose .csv file..." to select your prepared CSV
  4. Click "Upload CSV"
  5. Wait for processing — a progress indicator will show status

CSV Format Requirements

ColumnRequiredDescription
dc_page_idYesThe old Data Center content/page ID
cloud_page_idYesThe new Cloud content/page ID
space_keyOptionalConfluence space key for filtering
page_titleOptionalPage title for easier searching
ℹ File Size

Large files (100K+ rows) may take a few minutes to process. You'll receive a confirmation once complete.

Docs›Confluence Mapping›What to Expect

What to Expect After Upload

Understanding results after your mapping data is uploaded and processed.

Processing Results

Each row is categorized into one of two statuses:

  • Found — The DC page ID has a matching Cloud page. Users can click through directly.
  • Missing — No Cloud equivalent. The page didn't migrate, was deleted, or mapping data is incomplete.
💡 Tip

High percentage of "Missing"? Check CSV formatting or request an updated mapping file from Atlassian for migrations that completed after your initial file.

Admin Guide — Dashboard & Controls

Manage mapping data, use filters, view the dashboard, and control data lifecycle.

📸 Dashboard — Real-time overview of mapping statistics and search activity

Dashboard

The admin dashboard provides a real-time overview of your mapping data: total mappings, searches in the last 7 days, successful lookups, not-found counts, all-time statistics, and overall success rate. It also shows top searched terms and recent activity.

Not Found Tracker

📸 Not Found Tracker — Track pages users search for but can't find

The Not Found Tracker shows pages that users frequently search for but can't find in your mapping data. Use this to identify gaps in your migration and prioritize remediation.

Filter Options

  • By Space Key — Filter to specific Confluence space
  • By Status — Show only "Found" or "Missing" links
  • By Search — Search by DC page ID, Cloud ID, or page title

Data Deletion

📸 Privacy & Data — Danger Zone for permanent data deletion with audit log

Navigate to Privacy & Data in the admin sidebar. The Danger Zone section allows you to permanently delete all uploaded files, processed chunks, search index, and metadata. Every deletion is recorded in the Deletion Audit Log with timestamp and actor ID.

⚠ Deletion is Permanent

This action cannot be undone. Export any data you need before deleting. The deletion event will be recorded in the audit log.

User Guide — Search & Export

How to use the Link Lookup feature to find old DC links and export results.

📸 Link Lookup — The user-facing search interface for finding migrated Confluence pages

Single Lookup

Open the Link-a-Link panel and you'll see the Link Lookup interface. Select the Single tab, then paste an old DC URL, Page ID, or page title into the search box. Click Search to get the new Cloud URL instantly.

Bulk Check

Switch to the Bulk Check tab to look up multiple DC links at once. Paste multiple DC page IDs (one per line) and the system will process all entries, displaying the DC ID, Cloud link, and status for each.

CSV Export

From the results, you can export all displayed results as a CSV file. This is useful for sharing migration audit reports, tracking links that need manual remediation, or importing into project management tools.

How to Use (Quick Reference)

  1. Paste an old DC URL, Page ID, or title
  2. Click Search or use Bulk Check
  3. Get the new Cloud URL instantly

IntellinQs (Smart URLs)

Configure dynamic link systems and generate document URLs that update everywhere when the base URL changes.

📸 IntellinQs — Configure systems with name, prefix, and base URL

What is IntellinQs?

IntellinQs lets admins define a system name, prefix, and base URL (e.g., SmartSolve + PDL + https://smartsolve.app/docs/). Content authors then use a Confluence macro to insert a prefix and document ID. The full URL is constructed dynamically on every page render.

If the base URL ever changes, the admin updates it once — and every page using that system's prefix automatically reflects the new URL. No manual page edits required.

Key Benefits

  • Single source of truth for external system URLs
  • Zero manual updates when URLs change
  • Consistent link patterns across all Confluence content
  • Systems organized in a clean folder structure
  • Link Preview Generator to test URLs before publishing
  • Future-proof against the next migration or domain change
Docs›IntellinQs›Adding Systems

Adding Systems & URLs

Admin guide to configuring new systems, prefixes, and base URLs.

📸 Add New System — Enter system name, prefix, and base URL

Adding a New System

  1. Open the Link-a-Link Admin Panel
  2. Click IntellinQs in the left sidebar
  3. Under "Add New System", fill in:
    • System Name — e.g., SmartSolve, SAP, Pola (any format: uppercase, lowercase, mixed)
    • Prefix — e.g., PDL (any characters, including dashes)
    • Base URL — e.g., https://smartsolve.app/docs/ (trailing slash added automatically)
  4. Click "Add System"

The system will appear under Configured Systems above the form. You can also use the Link Preview Generator at the bottom to test any document number against your configured systems.

Folder Organization

How systems and URLs are organized in the admin panel.

How Folders Work

Each unique system name creates a folder in the Configured Systems section. When you add another URL for the same system name, it is grouped under the existing folder — no duplicate folder is created.

Docs›IntellinQs›Changing URLs

Changing Base URLs

How to update a system's base URL and what happens across all Confluence pages.

Updating a Base URL

  1. Open IntellinQs in the admin panel
  2. Find the system under Configured Systems
  3. Click the edit action on the system entry
  4. Enter the new base URL
  5. Save changes

What Happens Next

Every Confluence page using an IntellinQs macro with that system's prefix will automatically resolve to the new URL on the next page view. The URL is constructed dynamically at render time — no background job, no delay.

💡 Example

Change SmartSolve's base from https://old.smartsolve.com/docs/ to https://new.smartsolve.io/docs/ — every page with the PDL prefix (PDL-123, PDL-456, etc.) instantly points to the new domain.

⚠ Verify Before Changing

This change affects all pages using this prefix. Verify the new URL is correct and accessible before saving.

Docs›IntellinQs›Macro

Using the IntellinQs Macro

How content authors insert dynamic Smart URLs into Confluence pages.

Inserting a Smart URL

  1. Open any Confluence page in edit mode
  2. Type /link-a-link or search for "Link-a-Link" in the macro browser
  3. Select the IntellinQs macro
  4. Choose the System from the dropdown (e.g., SmartSolve)
  5. Enter the Document ID (e.g., PDL-123)
  6. Optionally set a display label
  7. Click Insert
ℹ No Re-edit Needed

You do not need to re-edit pages when a base URL changes. The macro resolves the URL at view time, so your page always shows the current correct URL.

Docs›Privacy›Overview

Data Handling Overview

How Link-a-Link handles data, what we store, and what we don't.

What We Store

  • Mapping data (DC page IDs + Cloud page IDs) — uploaded by admin
  • IntellinQs configurations (system names, prefixes, base URLs) — configured by admin

What We Do NOT Access

  • Confluence page content — never reads or processes your pages
  • User personal data — no names, emails, or identifiable information
  • Analytics or tracking data — no user behavior is tracked
ℹ GDPR & Data Residency

All data hosted in EU-region infrastructure. See the full Privacy Policy for details.

Docs›Privacy›Deleting Data

Deleting Your Data

How admins can permanently delete all Link-a-Link data from the system.

📸 Privacy & Data — Danger Zone with permanent deletion and audit log

Steps

  1. Open the Link-a-Link admin panel
  2. Click Privacy & Data in the sidebar
  3. Expand the Danger Zone — Data Management section
  4. Click "Delete All Data"
  5. Confirm in the dialog
⚠ This Cannot Be Undone

Permanently removes all uploaded files, processed chunks, search index, and metadata from the server. The deletion event will be recorded in the audit log with your account details and timestamp.

Deletion Audit Log

Every deletion is permanently logged with a timestamp and actor ID. This ensures a complete audit trail even after data is removed.

Docs›Support›Report

Generating a Support Report

The plugin generates a diagnostic report to help our engineers troubleshoot issues quickly.

📸 Support — Plugin health, configuration snapshot, and error logs with expandable sections

How to Generate

  1. Open the Link-a-Link admin panel
  2. Click Support in the sidebar
  3. Click the "Generate Support Report" button
  4. The report is generated on-demand and includes expandable sections
  5. Download the JSON file
  6. Attach it to your support ticket

Report Contents

The report contains only technical diagnostic data — no consumer data, no page content. It's generated on-demand and not stored. Sections include:

  • Tenant & Plugin Info — Instance ID, plugin version, environment
  • Dataset Status — Mapping data statistics
  • Settings Snapshot — Current configuration
  • Search Statistics — Aggregate data, no individual queries
  • Upload Error Log — Any errors during CSV uploads
  • UX Issue Log — Plugin errors only
  • Audit Log — Admin actions log
ℹ Privacy

The report is in JSON Format, Generated On Demand (not stored), and contains No Consumer Data. These badges are visible at the top of the Support page.
