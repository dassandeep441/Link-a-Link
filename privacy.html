Privacy Policy

How Link-a-Link handles your data. Built with privacy-first architecture for enterprise organizations.

Last updated: April 2026

Link-a-Link is a Confluence Cloud plugin developed by Agilva Solutions. This policy describes what data we collect, how we use it, and how you can control it.

Summary: Link-a-Link does not access your Confluence page content. We do not collect personal user data. We do not track user behavior. All data is admin-controlled and can be deleted at any time.

1. Data We Collect

Link-a-Link only stores data that you (the admin) explicitly provide:

Data TypeSourcePurpose
Mapping RecordsUploaded CSVDC-to-Cloud link resolution
Smart URL ConfigsAdmin inputDynamic URL generation
Confluence Instance IDAtlassian ForgeMulti-tenant isolation

2. Data We Do NOT Collect

  • Confluence page content (titles, body text, attachments)
  • User personal information (names, emails, avatars)
  • User behavior or analytics data
  • Browser fingerprints or cookies
  • IP addresses or location data

3. Data Storage & Hosting

All Link-a-Link data is stored within the Atlassian Forge platform infrastructure, hosted in EU-region data centers. We do not maintain separate databases or transfer data to external services.

4. GDPR Compliance

Link-a-Link's data handling practices are aligned with GDPR requirements:

  • Data Minimization: We only store what's necessary for functionality
  • Purpose Limitation: Data is used only for link mapping and URL generation
  • Right to Erasure: Admins can delete all data at any time
  • Transparency: This policy fully describes all data practices

5. Data Deletion

Admins can permanently delete all Link-a-Link data at any time through the plugin's admin panel (Settings → Data Management → Delete All Data). Deletion is immediate and irrecoverable. Once deleted, no records remain in our storage.

6. Third-Party Access

Link-a-Link does not share, sell, or transfer any data to third parties. No external APIs, analytics services, or advertising networks receive any data from the plugin.

7. Security

Link-a-Link is built on Atlassian Forge, which provides enterprise-grade security including encrypted data at rest and in transit, role-based access controls, and Atlassian's security review process.

8. Contact

For privacy-related questions or data requests, contact us through our Support Portal or email us at privacy@agilvasolutions.com.
